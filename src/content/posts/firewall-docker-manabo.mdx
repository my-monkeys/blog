  cause="Caddy (en container Docker) atteint l'hôte via host.docker.internal, mais UFW est en default deny et bloque les paquets venant du range 172.16.0.0/12 (réseau Docker)."
  fix="Ajouter une règle UFW : sudo ufw allow from 172.16.0.0/12 to any port 3456 proto tcp comment 'manabo from Docker'"
/>

## Lessons

À chaque fois qu'un service host-direct (pas dans Docker) doit être atteint par Caddy, **vérifier UFW avant tout**.
